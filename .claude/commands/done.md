# Завершение сессии разработки

Выполни шаги по порядку. Не спрашивай пользователя, что было сделано: сам восстанови итог сессии по диалогу, `git status`, `git diff`, измененным файлам и выполненным проверкам.

## 1. Сформируй итог сессии

Коротко перечисли, что реально было сделано:

- какие файлы созданы, изменены или удалены;
- какие баги исправлены;
- какие проверки запускались и чем закончились;
- что осталось незавершенным или заблокированным.

Если часть изменений была сделана до текущего агента и происхождение неясно, явно пометь это как существующее изменение и не приписывай себе.

## 2. Обнови TASKS.md

- Найди текущий день проекта.
- Отметь выполненные пункты как `✅`.
- Начатые, но не завершенные пункты отметь как `🔄`.
- Не начатые пункты оставь как `📋`.
- Если фактический прогресс отличается от старого описания, исправь текст задачи.

## 3. Полностью обнови AGENT_HANDOFF.md

Файл должен быть самодостаточным и актуальным:

- текущий день и статус;
- что сделано в этой сессии;
- какие файлы изменены;
- какие проверки пройдены;
- последняя ошибка или блокер;
- следующий конкретный шаг с командами;
- важный контекст для следующего агента.

## 4. Проверь состояние проекта

Запусти релевантные проверки для сделанных изменений. Минимум:

```bash
git status --short
```

Для n8n JSON:

```bash
node -e "const fs=require('fs'); const wf=JSON.parse(fs.readFileSync('n8n/Telegram Bot - Command Router.json','utf8')); for (const n of wf.nodes) if (n.parameters?.functionCode) new Function(n.parameters.functionCode); console.log('workflow ok')"
```

Для новых JS-файлов:

```bash
node --check scripts/telegram/command.js
```

## 5. Сделай git commit

Добавь в commit только релевантные файлы этой сессии. Не добавляй `.env`, секреты, локальные настройки и чужие unrelated changes.

Формат сообщения:

```text
Day X: краткое описание сделанного
```

Если commit невозможен или небезопасен из-за чужих изменений, не форсируй: объясни причину в финальном ответе.

## 6. Финальный ответ

Скажи пользователю:

- что сессия закрыта;
- что было сделано;
- какие проверки прошли;
- был ли сделан commit;
- какой следующий шаг.

Заверши фразой:

```text
Сессия закрыта. В следующий раз напиши /start для продолжения.
```
