# План: «Пульт» Tier 4 — UI-first (ядро Фазы 2)

## Context (зачем)

Пользователь уточнил: дашборд должен быть **ежедневным «пультом» обработки лидов для Tier 4**, а не страницей статистики. Текущий Обзор «вообще не то что нужно». Решение пользователя: **строим UI сейчас** (данные/результаты не нужны), а реальную генерацию и рассылку подключаем потом — **Claude API для писем/аудита, Resend для отправки**.

Ключевой факт: **схема БД уже поддерживает весь воркфлоу** (`prisma/schema.prisma`):
`LeadDelivery` (delivered_at → сегодняшние лиды; lead_outcome sent→in_progress→won/no_response/lost; deal_value),
`GeneratedContent` (audit_text + email_text + status draft→confirmed→sent), `Subscriber` (tier, selected_niches, profile).
Не хватает только **воркеров**, которые наполняют эти таблицы — их делаем следующей фазой.

## Решения (зафиксированы)
- **UI-first.** Этот заход — только фронт «пульта». Сегодняшние лиды читаем из существующего пула обогащения
  (`enrichment` A/B/C), статусы/генерация/отправка — **кнопки на месте, но застаблены** («скоро», disabled/тост).
  Когда появятся воркеры — переключаем источник на `LeadDelivery`/`GeneratedContent` и активируем кнопки.
- **Без миграций БД** в этой фазе (чистый фронт на текущем VPS-монолите; автодеплой работает).
- Генерация позже — **Claude API** (`claude-sonnet-4-6`, литовский), отправка — **Resend**. Себестоимость замерить.
- **Разнос на Vercel+API (B2) — после пульта.**
- Акцент — тёплый янтарь (уже сделано), бренд `leads.webvibe`, тёмный sidebar.

## Информационная архитектура (меню)
`Обзор (пульт)` · **`Лиды`** (только выданные/рабочие) · **`Проверка`** (ревью аудита+письма) · **`База`** (ВСЕ лиды + выбор по нише + срез) · **`Аналитика`** (результативность работы) · `Письма`(скоро) · `CRM`(скоро) · `Настройки`.

> **«Ниши»** убираются как отдельный пункт → выбор/фильтр по нише и срез базы (ниши, кредит-риск) живут внутри **«База»**.
> **«Аналитика» = только результативность РАБОТЫ** (обработано/отправлено/ответы/сделки). «Аналитика базы» (срез) отдельной
> вкладкой не нужна — лёгкий срез показываем в «База».

> Ключевое разделение (уточнено пользователем): **«Лиды» = выданные нам** (рабочий набор на обработку, с комментариями по
> каждому, чтобы не терять/не забыть клиента). **«База» = вся база A/B/C с фильтрами** (Tier-4/админ-вид «видеть всё»).
> Старая «Очередь» по `needs_review` убирается — термин путал («что подтверждать?»). **«Проверка»** показывает ТОЛЬКО
> лиды, у которых сгенерированы аудит/письмо и они **ждут ревью человеком** (`GeneratedContent.status='draft'`), не все лиды.
> Сейчас пусто (генерации ещё нет) — честный empty-state, структура готова.

### 1. Обзор = «пульт» (главный экран)
Сверху вниз:
1. **Напоминания «Что сделать»** (верхний блок) — задачи, выведенные из статусов:
   «Обработать N сегодняшних лидов», «Рассмотреть M в Ревью», «Перезвонить» (список лидов с телефоном).
   Реальные per-lead напоминания (КП/перезвон по исходу) дозаполнятся, когда появятся `LeadDelivery.lead_outcome`.
2. **Карточка аналитики** с переключателем периода **день / неделя / месяц / квартал / год**, два блока:
   - *Обработка:* обработано · писем выслано · аудитов проведено.
   - *Результаты:* ответили · не ответили · сделки (+ сумма сделок).
   Живые цифры где есть (обработано — `getProcessedSince`), остальное «—/скоро» до воркеров.
3. **Сегодняшние лиды** — выданные сегодня ~N по тарифу (Tier 4 ≈ 30), из **разных ниш**. По каждому:
   компания · ниша · город · кредит-риск · **статус обработки** · кнопка **«Обработать»** (одна кнопка =
   аудит → отчёт по аудиту → письмо с предложением; пока застаблена) + быстрые действия (позвонить/карточка).

### 2. Лиды (карточка)
В `/leads/[id]` добавить блок **«Обработка»**: статус (не обработан / аудит готов / письмо-черновик / отправлено / ответ / сделка),
следующее действие, и **поле комментария** (что я напишу). UI сейчас, persistence — позже через `LeadDelivery`.

### 3. Проверка (замена «Очереди»)
Только лиды, у которых сгенерированы **аудит + письмо** и они ждут ревью (`GeneratedContent.status='draft'`) — НЕ все лиды.
По каждому: «Рассмотреть аудит» / «Рассмотреть письмо» → **«Подтвердить отправку»** (GDPR-предохранитель). Маленькая
**карточка «Ждут подтверждения»** — имена лидов + кнопки (подтвердить / рассмотреть аудит / письмо).
Сейчас — честный empty-state (контента ещё нет), структура готова.

## Технический подход (этот заход = только фронт)
- **Данные** (`src/lib/dashboard-queries.ts`, server-only Prisma):
  - `getTodaysLeads(limit)` — детерминированный дневной срез из пула `enrichment` (A/B/C, разные ниши); позже → `LeadDelivery` по `delivered_at`.
  - `getReminders()` — производный список задач (обработать сегодняшние, ревью, перезвонить).
  - `getCockpitAnalytics(period)` — обработано (`getProcessedSince`), писем/аудитов (`GeneratedContent.count` → 0),
    ответы/сделки (`LeadDelivery` по `lead_outcome` → 0). Структура готова, заполнится сама.
  - `getReviewDrafts()` — `GeneratedContent` status=draft (пока пусто).
  - переиспользовать `getQueueCount`, `getLeads`, `getCompanyDetail`, `getProcessedSince`, `getNicheStats`.
- **Компоненты** (`src/components/`): `cockpit-reminders`, `analytics-card` (период + 2 блока), `today-leads`
  (строка лида + `lead-action-button`), `lead-action-button` (client, stub → тост «подключится с генерацией»),
  `review-list`, `confirm-card`, `lead-notes` (client, stub). Переиспользовать `period-filter` (расширить день/неделя/квартал/год),
  `results-funnel`, `stat-card`, badges, ui/*.
- **Маршруты / переименования** (обновить `src/lib/nav.ts`):
  - `(dash)/page.tsx` — пульт (переписать: напоминания + аналитика-период + сегодняшние лиды).
  - **новый** `(dash)/leads` = выданные/рабочие лиды (с комментариями) — текущий список полной базы переезжает в `(dash)/base`.
  - `(dash)/base` — ВСЯ база + фильтр по нише + лёгкий срез (поглощает `(dash)/niches`, который удаляем).
  - `(dash)/queue` → `(dash)/review` = «Проверка» (только `GeneratedContent.status='draft'`).
  - `(dash)/analytics` = результативность работы (срез базы убрать в `/base`).
  - `(dash)/leads/[id]` += блок «Обработка» (статус + следующее действие + комментарий).
- **Застабленные действия** (явно помечены, активируются в следующей фазе):
  кнопка «Обработать» → audit-gen + email-gen (Claude); «Подтвердить отправку» → Resend; persistence комментариев/статусов → `LeadDelivery`.

## Фазы
- **Фаза A (этот план):** UI пульта — Обзор (напоминания + аналитика-период + сегодняшние лиды), Ревью, блок «Обработка» в карточке.
  Реальные лиды из пула, действия застаблены. Codex-гейт → деплой.
- **Фаза B (следующая):** воркеры — seed подписчика tier=4; `lead-select` (#14) → `LeadDelivery`; `audit-gen` (#7, PageSpeed + текст);
  `email-gen` (#15, Claude литовское письмо). Переключить источники на `LeadDelivery`/`GeneratedContent`, активировать «Обработать».
- **Фаза C:** Resend-отправка + исходы (`lead_outcome`) + живая аналитика результатов; настройка ключей (Claude, Resend) в Настройках.
- **Позже:** разнос на Vercel(UI) + тонкий API(VPS).
